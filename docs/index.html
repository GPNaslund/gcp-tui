Address map

The address
is the warning.

Each environment gets its own loopback address and port, so the address itself tells you where you are: staging on 127.0.0.2, prod on 127.0.0.3. The interface colours everything to match — green for the environments that are safe to poke at, red for production.

There's no ambient default project and no guessing from your filesystem; the config file you declare is the only source of truth. Before binding, a pre-flight check refuses any slot that already has a listener on it.

Panel functions

Built to keep you on one screen.

An in-terminal cockpit

A two-pane view of environments and their profiles. Start background tunnels, watch their output stream in, and inspect databases, users, or backups without leaving the screen.

Tunnels that clean up

Proxies run as tracked background processes that are killed when you quit. They're session-scoped, so a normal exit never leaves a listener behind.

Safe for agents to drive

Read commands can emit --json, and --dry-run prints the exact gcloud call without running it. A prod write is impossible without a human at a terminal.

Secrets stay off your shell

Pull Secret Manager values into a gitignored .env; set new versions over stdin. Profile passwords live in your OS keyring, never in config.

Every command is printed first

Each external gcloud and proxy command is printed before it runs. Transparency is the point, because these commands can reach production.

It checks ADC, not just login

doctor validates your gcloud login and your Application Default Credentials, notices when either has expired, and offers to fix it.

Write interlock

An agent can't
touch prod.

Reads are always allowed. --yes authorizes a non-prod write when there's no terminal, but it never authorizes prod. The only way to mutate production is a human typing the environment's name at a real terminal, so an agent or a CI job simply can't.

The same rule guards every write: opening a tunnel, creating a backup, setting a secret.

Human · TTY Agent · no TTY
non-prod allowed --yes required
prod typed confirm refused

Refused is final. --yes doesn't apply to prod.
